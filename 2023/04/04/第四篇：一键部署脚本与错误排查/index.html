从零开始搭建 Hexo 博客（四）：一键部署脚本与常见错误排查

上一篇我们成功部署了博客，但每次都要输入 hexo clean && hexo g && hexo d 太麻烦！
本篇教你写一个一键部署脚本，并解决 90% 的 Hexo 部署问题。

一、创建一键部署脚本

1. 在博客根目录（D:\Blog\my-blog）新建文件：deploy.sh

2. 复制以下内容到文件中：

```bash
#!/bin/bash
echo “🚀 开始部署流程…”
echo “🧹 步骤 1: 清理缓存”
hexo clean

echo “📄 步骤 2: 生成静态文件”
hexo generate

echo “📤 步骤 3: 部署到 GitHub”
hexo deploy

echo “✅ 部署完成！请等待 1-3 分钟后访问：https://hyrx-tg.github.io"
💡 Windows 用户注意：
用 VS Code 创建文件（确保编码为 UTF-8）
右键文件 → Git Bash Here → 运行 chmod +x deploy.sh
3. 执行脚本：
bash

编辑

./deploy.sh
输入 Token 完成部署（密码框粘贴 Token，不是 GitHub 密码！）
